# Molo documentation

This directory contains the Molo user documentation source. The documentation
site is built with MkDocs Material and can be published by GitHub Pages or
ReadTheDocs.

Write Chinese documentation under `source/` and English documentation under
`source_en/`. Files generated by local preview or static builds are not part of
the public source tree.

## Local preview

```bash
cd Molo_Document
python -m venv .venv
source .venv/bin/activate
python -m pip install --upgrade pip
python -m pip install -r requirements-docs.txt
mkdocs serve -f mkdocs.yml
```

Open `http://127.0.0.1:8000/` after the server starts.

Preview the English site on another port:

```bash
mkdocs serve -f mkdocs.en.yml -a 127.0.0.1:8001
```

## Build

```bash
cd Molo_Document
rm -rf site
mkdocs build -f mkdocs.yml --strict
mkdocs build -f mkdocs.en.yml --strict
```

The Chinese static site is written to `site/`, and the English static site is
written to `site/en/`.

## Deployment

- ReadTheDocs uses the repository-level `.readthedocs.yaml`.
- GitHub Pages can use `.github/workflows/docs.yml`; enable Pages with
  "GitHub Actions" as the source in the repository settings. The Chinese site is
  published at the root path and the English site at `/en/`.
